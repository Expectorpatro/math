# GEMM

GEMM的全称是General Matrix Multiplication，也就是通用矩阵乘法。这里讨论最基本的形式：

```text
C = A B
```

其中，$A$是一个$M\times K$矩阵，$B$是一个$K\times N$矩阵，因此结果$C$是一个$M\times N$矩阵。三个矩阵底层都用一个一维`vector`存储，并且采用行主序（即矩阵同一行中的元素在内存中连续存放）。

## 朴素矩阵乘法

### 实现

最直接的矩阵乘法通常按照`ijk`顺序写：

```cpp
for (int i{0}; i < M; ++i)
{
    for (int j{0}; j < N; ++j)
    {
        for (int k{0}; k < K; ++k)
        {
            C[i * N + j] += A[i * K + k] * B[k * N + j];
        }
    }
}
```

这个写法的逻辑非常直观：固定C中的一个位置`C[i * N + j]`，然后让`k`从`0`遍历到`K - 1`，计算A的第`i`行和B的第`j`列的内积。

### 问题

上述`ijk`顺序对缓存不友好。问题主要出在最内层循环对B的访问上。在`ijk`顺序中，最内层循环变量是`k`。当`k`变化时，访问$B$的顺序是：

```cpp
B[0 * N + j]
B[1 * N + j]
B[2 * N + j]
B[3 * N + j]
```

这相当于在访问$B$的同一列。由于$B$是行主序存储的，相邻两次访问之间会跨过一整行，也就是跨过`N`个元素。因此，对$B$的访问不是连续的，而是跳跃式的，导致cache的空间局部性较差。

## IKJ

### 实现

针对`ijk`顺序导致的空间局部性差的问题，我们把循环顺序改成`ikj`：

```cpp
for (int i{0}; i < M; ++i)
{
    for (int k{0}; k < K; ++k)
    {
        for (int j{0}; j < N; ++j)
        {
            C[i * N + j] += A[i * K + k] * B[k * N + j];
        }
    }
}
```

`ikj`顺序的访问模式可以概括为：固定一个$A$元素，连续读取$B$的一行，连续更新$C$的一行。这个访问顺序更符合行主序vector的内存布局，所以cache利用率通常比`ijk`更高。

### 问题

`ikj`相比`ijk`已经改善了访问顺序，但是`ikj`只是把访问顺序变得更连续，并没有充分保证时间局部性。换句话说，`ikj`解决的是“访问是否连续”的问题，但还没有解决“连续访问的数据能不能留在cache里反复使用”的问题。

从内向外分析。首先，对于固定的`i`，$C$的第`i`行需要被所有`k`不断累加：

```cpp
C[i * N + j] += A[i * K + 0] * B[0 * N + j];
C[i * N + j] += A[i * K + 1] * B[1 * N + j];
C[i * N + j] += A[i * K + 2] * B[2 * N + j];
```

如果$C$的一整行很长，当程序更新到`C[i * N + j]`中比较靠后的元素时，这一行前面那些刚刚更新过的$C$元素可能已经被挤出cache了，等到$k$发生变化、程序重新回到这一行前面部分时，前面那些$C$元素可能已经不在cache里了，程序就必须重新从更慢的cache层级或者内存中把它们加载回来。

其次，$B$的第`k`行不只会被一个`i`使用，而是会被很多不同的`i`使用。比如：

```cpp
// i = 0 时会用 B 的第 k 行更新 C 的第 0 行
C[0 * N + j] += A[0 * K + k] * B[k * N + j];

// i = 1 时还会用同一行 B 更新 C 的第 1 行
C[1 * N + j] += A[1 * K + k] * B[k * N + j];

// i = 2 时还会继续用同一行 B 更新 C 的第 2 行
C[2 * N + j] += A[2 * K + k] * B[k * N + j];
```

在`ikj`顺序里，程序会先把`i = 0`对应的所有`k`循环跑完，然后才进入`i = 1`。等到下一个`i`再次需要$B$的第`k`行时，中间已经访问了大量其他$B$行，原来那一行$B$很可能早就被cache替换出去了。所以，虽然$B$的单次访问是连续的，但$B$跨不同`i`的复用没有被很好地保留下来。

## Block

### 实现

根据前面的论证可知时间局部性差的原因是因为$B$的大小可能会很大，于是很自然的我们就想到了可以把矩阵分块，将$B$切小：

```cpp
for (int kk{0}; kk < K; kk += Kb_size)
{
    int kEnd{std::min(kk + Kb_size, K)};
    for (int jj{0}; jj < N; jj += Jb_size)
    {
        int jEnd{std::min(jj + Jb_size, N)};
        for (int i{0}; i < M; ++i)
        {
            for (int k{kk}; k < kEnd; ++k)
            {
                for (int j{jj}; j < jEnd; ++j)
                {
                    C[i * N + j] += A[i * K + k] * B[k * N + j];
                }
            }
        }
    }
}
```

此时从`jj`的循环中可以发现此时我们对$A$进行了复用，当$M$过大时，`jj + 1`将无法复用$A$的对应元素，所以我们进行了如下的修改：

```cpp
for (int ii{0}; ii < M; ii += Ib_size)
{
    int iEnd{std::min(ii + Ib_size, M)};
    for (int kk{0}; kk < K; kk += Kb_size)
    {
        int kEnd{std::min(kk + Kb_size, K)};
        for (int jj{0}; jj < N; jj += Jb_size)
        {
            int jEnd{std::min(jj + Jb_size, N)};
            for (int i{ii}; i < iEnd; ++i)
            {
                for (int k{kk}; k < kEnd; ++k)
                {
                    for (int j{jj}; j < jEnd; ++j)
                    {
                        C[i * N + j] += A[i * K + k] * B[k * N + j];
                    }
                }
            }
        }
    }
}
```

请注意这里我们增加了跨行访问次数。

### 问题

下面的代码可以进行SIMD。

```cpp
for (int j{jj}; j < Jend; ++j)
{
    C[i * N + j] += A[i * K + k] * B[k * N + j];
}
```

## SIMD

### 实现

先给出SIMD的辅助函数。这里需要区分`float`和`double`两种情况。AVX中的`__m256`寄存器宽度是256 bit，一个`float`占32 bit，所以一个`__m256`可以同时存放8个`float`元素，`float`版本每次循环处理8个连续元素。同理，`double`版本每次循环处理4个连续元素。

```cpp
void avxSIMD(T const &a, T const *rhsRow, T *resultRow, int begin, int end)
{
    if constexpr (std::is_same<T, float>::value)
    {
        __m256 va{_mm256_set1_ps(a)};
        int j{begin};

        for (; j + 7 < end; j += 8)
        {
            __m256 vb{_mm256_loadu_ps(rhsRow + j)};
            __m256 vc{_mm256_loadu_ps(resultRow + j)};
            vc = _mm256_fmadd_ps(va, vb, vc);
            _mm256_storeu_ps(resultRow + j, vc);
        }

        for (; j < end; ++j)
            resultRow[j] += a * rhsRow[j];
    }
    else if constexpr (std::is_same<T, double>::value)
    {
        __m256d va{_mm256_set1_pd(a)};
        int j{begin};

        for (; j + 3 < end; j += 4)
        {
            __m256d vb{_mm256_loadu_pd(rhsRow + j)};
            __m256d vc{_mm256_loadu_pd(resultRow + j)};
            vc = _mm256_fmadd_pd(va, vb, vc);
            _mm256_storeu_pd(resultRow + j, vc);
        }

        for (; j < end; ++j)
            resultRow[j] += a * rhsRow[j];
    }
}
```

### 问题

SIMD让“一条指令处理多个数”，但是注意到：

```cpp
for (int i{ii}; i < iEnd; ++i)
{
    for (int k{kk}; k < kEnd; ++k)
    {
        T const a{A[i * K + k]};
        T const *rhsRow{B + k * N};
        T *resultRow{C + i * N};
        avxSIMD(a, rhsRow, resultRow, jj, jEnd);
    }
}
```

对于固定的$i$和固定的一段$j:j+4$，同一段$C$会在每一个$k$上被反复读取和写回：

$$
\begin{gather*}
    C_{i,j:j+4}\leftarrow C_{i,j:j+4}+A_{i,k}B_{k,j:j+4}, \\
    C_{i,j:j+4}\leftarrow C_{i,j:j+4}+A_{i,k+1}B_{k+1,j:j+4}, \\
    C_{i,j:j+4}\leftarrow C_{i,j:j+4}+A_{i,k+2}B_{k+2,j:j+4}
\end{gather*}
$$

一直继续到$k=kEnd-1$。在数学上，这些更新是正确的；但是在机器执行层面，如果每一次更新都调用`avxSIMD`，那么每一次$k$循环都会发生如下过程：

```text
load C(i,j:j+4) -> 做一次FMA -> store C(i,j:j+4)
load C(i,j:j+4) -> 做一次FMA -> store C(i,j:j+4)
load C(i,j:j+4) -> 做一次FMA -> store C(i,j:j+4)
...
```

这就是普通SIMD写法的第一个瓶颈：它提高了单次计算的宽度，却没有延长$C$在寄存器中的生命周期。更理想的执行方式是：先把一段$C$加载到SIMD寄存器中，然后沿着$k$方向不断累加，等整个$k$循环结束以后，再把最终结果写回内存。注意此时内层循环的访问顺序又从`ikj`退回到了`ijk`。

第二个问题是一个CPU核心往往有多个SIMD寄存器，以x86-64上的AVX2为例，CPU通常有16个256-bit的YMM寄存器。在普通SIMD写法中我们只使用了3个，$A,B,C$各占一个。

## Micro Kernel

Micro kernel通常会让$C$中的一个小矩形块长期停留在寄存器中，而不是每次只计算$C$中的一个向量，从而充分使用硬件提供的向量寄存器。这个小矩形块就被称为micro kernel，$r$表示一次计算$C$的多少行，$c$表示一次计算$C$的多少列。下面假设$r=4$，不固定$c$，但固定$C$的每一行用两个SIMD寄存器。这样的话我们就将$C$存储在了$4\times2=8$个寄存器中，$B$用了两个寄存器，而$A$只需一个寄存器。由于我们一次计算了多行$C$，同一段$B$可以同时服务多行$C$，因此$B$在SIMD寄存器中的复用性也得到了增强。

### 实现

```cpp
void avxMicroKernel(T const *A,
                    T const *B,
                    T *C,
                    int iBegin,
                    int kBegin,
                    int kEnd,
                    int jBegin,
                    int K,
                    int N) const
{
    if constexpr (std::is_same<T, float>::value)
    {
        __m256 c00{_mm256_loadu_ps(C + iBegin * N + jBegin)};
        __m256 c01{_mm256_loadu_ps(C + iBegin * N + jBegin + 8)};
        __m256 c10{_mm256_loadu_ps(C + (iBegin + 1) * N + jBegin)};
        __m256 c11{_mm256_loadu_ps(C + (iBegin + 1) * N + jBegin + 8)};
        __m256 c20{_mm256_loadu_ps(C + (iBegin + 2) * N + jBegin)};
        __m256 c21{_mm256_loadu_ps(C + (iBegin + 2) * N + jBegin + 8)};
        __m256 c30{_mm256_loadu_ps(C + (iBegin + 3) * N + jBegin)};
        __m256 c31{_mm256_loadu_ps(C + (iBegin + 3) * N + jBegin + 8)};

        for (int k{kBegin}; k < kEnd; ++k)
        {
            __m256 b0{_mm256_loadu_ps(B + k * N + jBegin)};
            __m256 b1{_mm256_loadu_ps(B + k * N + jBegin + 8)};

            __m256 a{_mm256_set1_ps(A[iBegin * K + k])};
            c00 = _mm256_fmadd_ps(a, b0, c00);
            c01 = _mm256_fmadd_ps(a, b1, c01);

            a = _mm256_set1_ps(A[(iBegin + 1) * K + k]);
            c10 = _mm256_fmadd_ps(a, b0, c10);
            c11 = _mm256_fmadd_ps(a, b1, c11);

            a = _mm256_set1_ps(A[(iBegin + 2) * K + k]);
            c20 = _mm256_fmadd_ps(a, b0, c20);
            c21 = _mm256_fmadd_ps(a, b1, c21);

            a = _mm256_set1_ps(A[(iBegin + 3) * K + k]);
            c30 = _mm256_fmadd_ps(a, b0, c30);
            c31 = _mm256_fmadd_ps(a, b1, c31);
        }

        _mm256_storeu_ps(C + iBegin * N + jBegin, c00);
        _mm256_storeu_ps(C + iBegin * N + jBegin + 8, c01);
        _mm256_storeu_ps(C + (iBegin + 1) * N + jBegin, c10);
        _mm256_storeu_ps(C + (iBegin + 1) * N + jBegin + 8, c11);
        _mm256_storeu_ps(C + (iBegin + 2) * N + jBegin, c20);
        _mm256_storeu_ps(C + (iBegin + 2) * N + jBegin + 8, c21);
        _mm256_storeu_ps(C + (iBegin + 3) * N + jBegin, c30);
        _mm256_storeu_ps(C + (iBegin + 3) * N + jBegin + 8, c31);
    }
    else if constexpr (std::is_same<T, double>::value)
    {
        __m256d c00{_mm256_loadu_pd(C + iBegin * N + jBegin)};
        __m256d c01{_mm256_loadu_pd(C + iBegin * N + jBegin + 4)};
        __m256d c10{_mm256_loadu_pd(C + (iBegin + 1) * N + jBegin)};
        __m256d c11{_mm256_loadu_pd(C + (iBegin + 1) * N + jBegin + 4)};
        __m256d c20{_mm256_loadu_pd(C + (iBegin + 2) * N + jBegin)};
        __m256d c21{_mm256_loadu_pd(C + (iBegin + 2) * N + jBegin + 4)};
        __m256d c30{_mm256_loadu_pd(C + (iBegin + 3) * N + jBegin)};
        __m256d c31{_mm256_loadu_pd(C + (iBegin + 3) * N + jBegin + 4)};

        for (int k{kBegin}; k < kEnd; ++k)
        {
            __m256d b0{_mm256_loadu_pd(B + k * N + jBegin)};
            __m256d b1{_mm256_loadu_pd(B + k * N + jBegin + 4)};

            __m256d a{_mm256_set1_pd(A[iBegin * K + k])};
            c00 = _mm256_fmadd_pd(a, b0, c00);
            c01 = _mm256_fmadd_pd(a, b1, c01);

            a = _mm256_set1_pd(A[(iBegin + 1) * K + k]);
            c10 = _mm256_fmadd_pd(a, b0, c10);
            c11 = _mm256_fmadd_pd(a, b1, c11);

            a = _mm256_set1_pd(A[(iBegin + 2) * K + k]);
            c20 = _mm256_fmadd_pd(a, b0, c20);
            c21 = _mm256_fmadd_pd(a, b1, c21);

            a = _mm256_set1_pd(A[(iBegin + 3) * K + k]);
            c30 = _mm256_fmadd_pd(a, b0, c30);
            c31 = _mm256_fmadd_pd(a, b1, c31);
        }

        _mm256_storeu_pd(C + iBegin * N + jBegin, c00);
        _mm256_storeu_pd(C + iBegin * N + jBegin + 4, c01);
        _mm256_storeu_pd(C + (iBegin + 1) * N + jBegin, c10);
        _mm256_storeu_pd(C + (iBegin + 1) * N + jBegin + 4, c11);
        _mm256_storeu_pd(C + (iBegin + 2) * N + jBegin, c20);
        _mm256_storeu_pd(C + (iBegin + 2) * N + jBegin + 4, c21);
        _mm256_storeu_pd(C + (iBegin + 3) * N + jBegin, c30);
        _mm256_storeu_pd(C + (iBegin + 3) * N + jBegin + 4, c31);
    }
}
```

```cpp
Matrix<T> matmulMicroKernelSIMD(Matrix<T> const &rhs) const
{
    checkMul(rhs);

    int const M{_rows};
    int const K{_cols};
    int const N{rhs._cols};
    Matrix<T> result{M, N, T{}};

    T const *A{&_data[0]};
    T const *B{&rhs._data[0]};
    T *C{&result._data[0];

    int const microRows{4};
    int const microCols{std::is_same<T, float>::value ? 16 : 8};

    for (int ii{0}; ii < M; ii += I_bSize)
    {
        int const iEnd{std::min(ii + I_bSize, M)};

        for (int kk{0}; kk < K; kk += K_bSize)
        {
            int const kEnd{std::min(kk + K_bSize, K)};

            for (int jj{0}; jj < N; jj += J_bSize)
            {
                int const jEnd{std::min(jj + J_bSize, N)};

                int i{ii};

                for (; i + microRows <= iEnd; i += microRows)
                {
                    int j{jj};

                    for (; j + microCols <= jEnd; j += microCols)
                    {
                        avxMicroKernel(A, B, C, i, kk, kEnd, j, K, N);
                    }

                    if (j < jEnd)
                    {
                        for (int row{i}; row < i + microRows; ++row)
                        {
                            T *resultRow{C + row * N};

                            for (int k{kk}; k < kEnd; ++k)
                            {
                                T const a{A[row * K + k]};
                                T const *rhsRow{B + k * N};

                                avxSIMD(a, rhsRow, resultRow, j, jEnd);
                            }
                        }
                    }
                }

                for (; i < iEnd; ++i)
                {
                    T *resultRow{C + i * N};

                    for (int k{kk}; k < kEnd; ++k)
                    {
                        T const a{A[i * K + k]};
                        T const *rhsRow{B + k * N};

                        avxSIMD(a, rhsRow, resultRow, jj, jEnd);
                    }
                }
            }
        }
    }

    return result;
}
```

### 问题

$A$和$B$矩阵在内层计算中都存在较严重的跨行访问问题，内存访问不够连续，缓存友好性较差。
